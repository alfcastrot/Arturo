# Arturo Knowledge Base

Base operativa compartida para Arturo, Hermes, Daniel y Mercurio.

## Modelo de capas

- `Layer 1`: evidencia cruda e inmutable.
- `Layer 2`: base viva de conocimiento derivado.
- `Layer 3`: reglas, limites y coordinacion de agentes.

## Como usarla

- `raw/` guarda fuentes sin procesar.
- `notes/` guarda notas limpias y conectadas.
- `summaries/` guarda resúmenes rapidos.
- `decisions/` guarda decisiones durables.
- `inbox/` guarda material pendiente.
- `log.md` registra operaciones cronologicas.
- `sources-map.md` registra procedencia y trazabilidad.

## Regla de trabajo

Si algo sirve mas de una vez, no se queda solo en el chat.
Se convierte en nota, decision o contexto reutilizable.

## Flujo recomendado

1. Capturar fuente o idea.
2. Resumir.
3. Extraer decisiones, riesgos y acciones.
4. Vincular con temas relacionados.
5. Reusar en futuras conversaciones.

## Agentes

- Arturo: coordina la operacion y decide donde guardar continuidad.
- Hermes: asesora, contrasta y detecta huecos o inconsistencias.
- Daniel: apoya la parte tecnica y de verificacion.
- Mercurio: usa la base para analisis y trading sin perder trazabilidad.

# Checklist Diario

## Antes de empezar

- Abrir `Arturo` en Obsidian.
- Verificar que estoy en la carpeta correcta.
- Revisar si hay tareas pendientes del dia.

## Durante el trabajo

- Escribir primero en la nota correcta.
- Guardar decisiones importantes en `00-Core` o `90-Logs`.
- Guardar comandos tecnicos en `06-Infra-Tecnica`.
- No mezclar material temporal con notas definitivas.

## Al cerrar

- Revisar cambios con `git status`.
- Si hay cambios utiles:
  - `git add .`
  - `git commit -m "<mensaje claro>"`
  - `git push`
- Confirmar que no quedaron archivos temporales fuera de lugar.

## Regla rapida

- Decision durable: se documenta.
- Comando util: se guarda.
- Error repetible: se anota.
- Ruido: se elimina o se deja en temporal.
